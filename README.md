## 🧭 Overview

**AgenticAI for BO** is an AI-powered tool designed to **read, parse, and analyze log files** generated by SAP Business Objects (BO). It identifies errors, warnings, and anomalies within the logs and **proposes actionable solutions** using Agentic AI capabilities.

Managing SAP Business Objects environments often involves sifting through large volumes of log files to diagnose issues. This project automates that process, saving administrators and support engineers significant time and effort.

### 🎯 Objectives

- **Automate log parsing** — Read and parse log files from SAP BO log directories.
- **Error detection** — Identify errors, exceptions, warnings, and critical failures.
- **Intelligent diagnosis** — Classify and categorize issues by severity and type.
- **Solution proposal** — Leverage AI to suggest resolutions based on known error patterns, SAP documentation, and historical fixes.
- **Reporting** — Generate structured reports summarizing findings and recommendations.

## 📌 Prerequisites

- **Linux server** (RHEL, CentOS, SUSE, Ubuntu, or any distro running SAP BO)
- **Python** 3.9 or higher
- **SAP Business Objects** log files (`.log`, `.glf`, `.trc` formats)
- **API Key** for the AI/LLM provider (e.g., OpenAI or Google Gemini) — *optional, templates work without AI*
- **pip** (Python package manager)
- **Read access** to the SAP BO log directory (e.g., `/opt/sap_bobj/enterprise_xi40/logging`)

## 🚀 Installation (Linux Server)

1. **Clone the repository**:

   ```bash
   git clone <repository-url>
   cd AgenticAI_for_BO
   ```

2. **Create a virtual environment**:

   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

3. **Install dependencies**:

   ```bash
   pip install -r requirements.txt
   ```

4. **Set up environment variables**:

   ```bash
   cp .env.example .env
   nano .env
   # Set SAP_BO_LOG_DIR to your actual log path
   # Add your AI API key (OpenAI or Gemini)
   ```

5. **Verify the log directory is accessible**:

   ```bash
   # Check the log directory exists and has log files
   ls -la /opt/sap_bobj/enterprise_xi40/logging/
   ```

## ⚙️ Configuration

Create a `.env` file or update `config.yaml` with the following settings:

### `.env` (API keys & paths)

```bash
# AI Provider
AI_PROVIDER=gemini
GEMINI_API_KEY=your-gemini-api-key

# SAP BO Log Directory (Linux path)
SAP_BO_LOG_DIR=/opt/sap_bobj/enterprise_xi40/logging

# Common Linux log paths:
#   /opt/sap_bobj/enterprise_xi40/logging
#   /usr/sap/bobj/enterprise_xi40/logging
#   /home/boadmin/sap_bobj/logging
```

### `config.yaml` (analysis settings)

```yaml
# SAP Business Objects Log Directory
log_directory: "/opt/sap_bobj/enterprise_xi40/logging"

# Log file extensions to process
log_extensions:
  - ".log"
  - ".glf"
  - ".trc"

# AI Provider Configuration
ai_provider: "gemini"          # Options: openai, gemini
ai_model: "gemini-pro"

# Output Settings
output_format: "markdown"      # Options: markdown, json, console
output_directory: "./reports"

# Analysis Settings
severity_filter: "WARNING"     # Minimum severity: INFO, WARNING, ERROR, CRITICAL
max_files: 50                  # Max number of log files to process per run
```

## 💻 Usage

### Basic Usage

```bash
# Activate virtual environment
source venv/bin/activate

# Analyze logs from the configured directory (reads SAP_BO_LOG_DIR from .env)
python main.py

# Specify a custom log directory
python main.py --log-dir /opt/sap_bobj/enterprise_xi40/logging

# Filter by severity
python main.py --severity ERROR

# Generate a Markdown report
python main.py --output markdown

# Generate a JSON report
python main.py --output json

# Combine options
python main.py --log-dir /var/log/sap_bobj --severity CRITICAL --output markdown
```

### Run as a Cron Job (Scheduled Analysis)

```bash
# Edit crontab
crontab -e

# Run analysis every day at 6 AM and save the Markdown report
0 6 * * * cd /path/to/AgenticAI_for_BO && /path/to/AgenticAI_for_BO/venv/bin/python main.py --output markdown >> /var/log/agenticai_bo.log 2>&1
```

### Run as a systemd Service (Optional)

Create `/etc/systemd/system/agenticai-bo.service`:

```ini
[Unit]
Description=AgenticAI for SAP Business Objects Log Analyzer
After=network.target

[Service]
Type=oneshot
WorkingDirectory=/path/to/AgenticAI_for_BO
ExecStart=/path/to/AgenticAI_for_BO/venv/bin/python main.py --output markdown
User=boadmin
Group=sapsys

[Install]
WantedBy=multi-user.target
```

```bash
# Enable and run
sudo systemctl daemon-reload
sudo systemctl enable agenticai-bo
sudo systemctl start agenticai-bo
```

## 📁 Supported Log Types

The tool is designed to process common SAP Business Objects log files, including but not limited to:

| Log File                  | Description                                      |
| ------------------------- | ------------------------------------------------ |
| `bia_cms_audit.log`       | Central Management Server (CMS) audit logs       |
| `bia_wcs.log`             | Web Component Server logs                        |
| `bia_frs.log`             | File Repository Server logs                      |
| `bia_aps.log`             | Adaptive Processing Server logs                  |
| `bia_webiserver.log`      | Web Intelligence Processing Server logs          |
| `crystal_reports.log`     | Crystal Reports processing logs                  |
| `ConnectionServer*.log`   | Connection Server logs (database connectivity)   |
| `TraceLog_*.glf`          | General trace/debug log files                    |

## 🔄 How It Works

1. **📂 Ingestion** — The tool scans the configured SAP BO log directory and collects all matching log files.
2. **🔎 Parsing** — Each log file is parsed line-by-line, extracting timestamps, severity levels, component names, and message content.
3. **⚠️ Error Detection** — Entries matching error/warning patterns are flagged and extracted for analysis.
4. **🧠 AI Analysis** — Flagged entries are sent to the configured AI model, which classifies the issue and generates a proposed solution.
5. **📝 Reporting** — Results are compiled into a structured report with errors, their context, and recommended solutions.
